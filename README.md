## ◈ Featured Projects

<details>
<summary><b>◆ Raseed — GST-Compliant Invoicing Platform for Indian SMBs</b></summary>

<br/>

> A privacy-first, offline-capable invoicing application engineered for Indian freelancers and small businesses. Built on Flutter with local AES-256 encrypted storage via Hive, and Firebase strictly scoped to authentication and subscription management. Targets full GST compliance and Razorpay payment integration.

<div align="center">

| Attribute | Detail |
|-----------|--------|
| **Stack** | Flutter · Dart · Hive · Firebase Auth · Razorpay · Cloud Functions |
| **Platform** | Android · Web (Phase 1 Launch Scope) |
| **Architecture** | Offline-first · Local encrypted storage · Event-driven billing |
| **Security** | AES-256 at rest · Zero raw PII on server · Privacy-first design |
| **Differentiator** | No cloud invoice storage — full local ownership for the user |
| **Market** | Indian SMB / Freelancer segment · GST-compliant invoicing |
| **Repository** | [github.com/abhinavtech23/raseed](https://github.com/abhinavtech23) |

</div>

Raseed addresses a critical gap in the Indian freelancer ecosystem: GST-compliant invoicing tools that don't compromise user data. Every invoice is encrypted and stored locally; Firebase touches only auth tokens and subscription state. The payment architecture is powered by Razorpay via Firebase Cloud Functions, keeping secrets off-device. Domain: `raseed.in`.

<br/>

</details>

<details>
<summary><b>◆ OUTRUN — GPS Territory-Capture Kingdom Builder</b></summary>

<br/>

> A real-time multiplayer GPS game for the Indian market combining H3 hexagonal territory capture with Clash of Clans-style isometric kingdom rendering overlaid on a live city map. Designed for outdoor engagement with a Snapchat Maps aesthetic and tactile UI interactions.

<div align="center">

| Attribute | Detail |
|-----------|--------|
| **Stack** | Flutter · Riverpod 2.x · Mapbox · H3 (Resolution 9) · Firebase Auth · Rive |
| **Rendering** | CustomPainter · Isometric projection · Real-time hex overlays |
| **Architecture** | Stream-based state · Phone OTP auth · Hex zone ownership engine |
| **Animation** | Rive flag animations · PlayfulButton haptic interactions |
| **Map Style** | Mapbox warm light · City-scale territory visualization |
| **Target** | Hyper-local Indian urban market · Outdoor social gameplay |
| **Repository** | [github.com/abhinavtech23/outrun](https://github.com/abhinavtech23) |

</div>

OUTRUN is engineered around H3 Resolution 9 hexagons — each cell covering approximately 0.1 km² — enabling fine-grained, real-world territory ownership. The rendering pipeline uses Flutter's CustomPainter with a hand-rolled isometric projection matrix, achieving smooth 60fps hex overlays atop live Mapbox tiles. Kingdom assets render as 3D isometric sprites reacting to ownership events in real time.

<br/>

</details>

<details>
<summary><b>◆ ESP8266 Smart Relay Controller — Embedded Web Server</b></summary>

<br/>

> A hardware-software integration project built on the ESP8266 NodeMCU platform, enabling relay control via a browser-hosted web server over Wi-Fi. Engineered to handle active-LOW/HIGH relay logic with correct hardware wiring and software debounce.

<div align="center">

| Attribute | Detail |
|-----------|--------|
| **Stack** | C++ (Arduino) · ESP8266 · NodeMCU · HTML/CSS Web UI |
| **Protocol** | HTTP over Wi-Fi · ESP8266WebServer |
| **Domain** | Embedded Systems · IoT · Hardware-Software Integration |
| **Challenge** | Active-LOW relay logic debugging · GPIO state management |
| **Application** | Home automation · Remote switching · IoT control plane |
| **Repository** | [github.com/abhinavtech23](https://github.com/abhinavtech23) |

</div>

This project demonstrates hardware-level systems thinking: correctly mapping GPIO states to active-LOW relay modules requires understanding the inversion between software logic and physical switching behavior — a detail that separates embedded practitioners from surface-level IoT tinkerers.

<br/>

</details>
